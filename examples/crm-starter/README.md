# Example: CRM Starter (stub)

Unlike `booking-system/`, this one isn't pre-filled — it's a starting prompt so you can watch the full pipeline run on a second idea, faster the second time.

## Idea

A minimal CRM: track contacts, log a note per interaction, and see who hasn't been followed up with in 2+ weeks.

## Run it

```
Build an MVP: a minimal CRM — track contacts, log a note per interaction, and
show who hasn't been followed up with in 2+ weeks.

Use this repository. Follow the full workflow (research → prd → roadmap → ui →
build → debug → deploy) and save the output of each stage into this folder,
the same way examples/booking-system/ is structured.
```

Compare your PRD and roadmap against `../booking-system/` once you're done — same pipeline, different idea, and the scope decisions should look different because the problem is different.

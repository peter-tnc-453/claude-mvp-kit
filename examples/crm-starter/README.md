# Example: CRM Starter (stub)

Unlike `booking-system/`, this one isn't pre-filled — it's a starting prompt so you can watch the full pipeline run on a second idea, faster the second time.

## Idea
A minimal CRM: track contacts, log a note per interaction, see who you haven't followed up with in 2+ weeks.

## Run it

```
สร้าง MVP: CRM ขั้นต่ำ — เก็บรายชื่อผู้ติดต่อ, บันทึกโน้ตการคุยแต่ละครั้ง,
ดูว่าใครที่ยังไม่ได้ follow up เกิน 2 สัปดาห์

ใช้ repository นี้ ทำตาม workflow เต็ม (research → prd → roadmap → ui → build →
debug → deploy) เก็บผลลัพธ์แต่ละขั้นตอนไว้ในโฟลเดอร์นี้เหมือน examples/booking-system/
```

Compare your PRD and roadmap against `../booking-system/` once you're done — same pipeline, different idea, and the scope decisions should look different because the problem is different.
